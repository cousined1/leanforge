## SEO requirements

### 1. Core technical SEO
Implement or verify:
- Unique `<title>` and meta description for every indexable page.
- Canonical URL tags on all canonical pages.
- Clean heading hierarchy with exactly one meaningful H1 per page.
- Open Graph and Twitter/X meta tags for major public pages.
- Sitemap generation or sitemap.xml support.
- robots.txt with sensible crawl directives.
- Internal links between marketing pages, legal pages, and high-value product pages.
- Descriptive anchor text, not generic “click here”.
- Clean semantic HTML and accessible navigation.
- Fast-loading pages, lazy-loaded non-critical images, explicit width/height on media, and no broken links. [perplexity](https://www.perplexity.ai/search/bdaf109e-6af1-490c-9c81-dc9e0ae8ac50)

### 2. JSON-LD structured data
Use JSON-LD in the `<head>` or equivalent framework metadata implementation. Google recommends JSON-LD for structured data, so prefer it over microdata unless the stack forces otherwise. [perplexity](https://www.perplexity.ai/search/69232a80-3c55-4fc7-9297-07f45ad4672c)

Implement the structured data types that fit the site:

#### Always include when applicable
- Organization
- WebSite
- WebPage
- BreadcrumbList
- AboutPage for About page
- ContactPage for Contact page [perplexity](https://www.perplexity.ai/search/69232a80-3c55-4fc7-9297-07f45ad4672c)

#### Add when applicable to the business model
- SoftwareApplication for SaaS/app product pages.
- Product for productized software pages with pricing/features.
- FAQPage only where there is a real visible FAQ section on the page.
- Article or BlogPosting for articles/resources.
- LocalBusiness only if the site is truly representing a local business footprint in a way that matches public business details.
- Service for service pages if the site sells services rather than software. [perplexity](https://www.perplexity.ai/search/69232a80-3c55-4fc7-9297-07f45ad4672c)

### 3. Rich snippets support
Add structured data only where the on-page visible content supports it. Do not spam schema types that do not match the rendered page. Rich snippet eligibility should be earned through accurate markup and corresponding visible content. [perplexity](https://www.perplexity.ai/search/69232a80-3c55-4fc7-9297-07f45ad4672c)

Examples:
- FAQ schema only on pages with visible FAQ questions and answers.
- Breadcrumb schema only where breadcrumb navigation or clear site hierarchy exists.
- SoftwareApplication schema on product/app pages with clear app name, description, operating context, offers if available, and publisher/organization.
- Review/AggregateRating schema only if the site genuinely displays compliant first-party review content. Do not fabricate ratings. [perplexity](https://www.perplexity.ai/search/69232a80-3c55-4fc7-9297-07f45ad4672c)
